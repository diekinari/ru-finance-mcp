# news.md — новостной слой FinAgent (плейбук для агента)

Новости **не** в MCP-сервере `ru-finance` — это намеренно. Их собирает **сам агент**
через свой веб-поиск/веб-fetch. Этот файл — инструкция: куда ходить, как
фильтровать и как привязывать к портфелю.

> Данные по 2026 году бери ТОЛЬКО из живых источников/поиска. Не отвечай по памяти —
> у LLM знания устаревшие. Если что-то не нашлось — так и скажи.

---

## Правила работы с новостями

1. **Помечай каждый пункт:** `[ФАКТ]` (событие/число) · `[ОЖИДАНИЕ]` (прогноз/ещё не принято) · `[МНЕНИЕ]` (трактовка аналитика).
2. **Рефлексивность:** рынок торгует ОЖИДАНИЯМИ, а не событиями. Новость объясняет/контекстуализирует, но не предсказывает.
3. **Несколько источников:** не доверяй одному; отделяй факт от сентимента.
4. **Взвешивай источник:** у части изданий редполитика/особый правовой статус (иноагенты), у брокеров — конфликт интересов. Помечай «мнение», не «факт».
5. **Всегда привязывай к портфелю:** не «вышла новость», а «какое ожидание сместилось → по теории какие бумаги чувствительны → что это значит для этих позиций».
6. **Цепочка вывода:** новость/сигнал → сместившееся ожидание/режим → стандартная теория → чувствительные инструменты → конкретные позиции портфеля.

## Источники по режиму потребления

### Уровень 1 — авто-поток (машиночитаемые RSS, доступны, проверено 27.06.2026)
Это ядро. Тяни RSS напрямую, фильтруй за нужный период.

| Источник | URL | Для чего |
|---|---|---|
| **ЦБ — пресс-релизы** | `https://www.cbr.ru/rss/RssPress` | **решения по ставке**, пресс-релизы (главное) |
| **ЦБ — события** | `https://www.cbr.ru/rss/eventrss` | инфляц. ожидания, обзоры, аналитика ЦБ |
| **РБК** | `https://rssexport.rbc.ru/rbcnews/news/30/full.rss` | оперативка по рынку/экономике |
| **ПРАЙМ** | `https://1prime.ru/export/rss2/index.xml` | экономика, нефть, санкции |
| **Интерфакс** | `https://www.interfax.ru/rss.asp` | оперативка (отдаёт 301 → следуй редиректу) |
| **Финам** | `https://www.finam.ru/analysis/conews/rsspoint/` | рынок акций/облигаций, RGBI, дивиденды |
| **Smart-Lab** | `https://smart-lab.ru/news/rss/` | рыночный поток, второй эшелон |
| **Ведомости** | `https://www.vedomosti.ru/rss/news` | аналитика, корп. новости (частично пейволл) |

Совет: 2–3 канала в ядро (ЦБ + РБК/ПРАЙМ), остальное — точечно. Больше = шум и дубли.

### Уровень 2 — агент по запросу (веб-поиск, когда RSS не хватает)
То, что блокирует ботов / за пейволлом / формируется вовне — только через веб-поиск, на уровне заголовков/сводок.

| Тема | Источники | Заметка |
|---|---|---|
| Внешний взгляд, геополитика, потоки капитала | **Reuters / FT / Bloomberg** | блокируют автозагрузку (401/403) — только поиском, полные тексты закрыты |
| **Нефть Brent** | EIA `https://www.eia.gov/dnav/pet/hist/RBRTEd.htm`, Trading Economics | EIA доступен; Brent — макродрайвер №1 вместе со ставкой |
| **Дисконт Urals** | Минфин (для налога), Reuters/Moscow Times | публикуется нерегулярно; важен для бюджета/экспортёров |
| Деловой РФ-контекст | The Bell `https://thebell.io`, Frank Media `https://frankmedia.ru` | доступны; часть со статусом иноагента — взвешивать |

### Уровень 3 — событийно/точечно (под конкретную бумагу)
| Источник | URL | Для чего |
|---|---|---|
| **Раскрытие эмитентов** | `https://www.e-disclosure.ru` | существенные факты, отчётность, дивиденды, ГОСА |
| Telegram брокеров/аналитиков | — | быстрее всех, но шум; только как **сигнал**, не факт |
| Брокерская аналитика | Сбер/Т/БКС/Альфа/Финам | таргеты/прогнозы — всегда метка «мнение», конфликт интересов |

### Что НЕ тянуть автоматически (проверено — не выходит)
- **Полные санкционные списки** OFAC (таймаут) / UK OFSI (16 МБ). Вместо этого — **точечно**: «затронуты ли мои эмитенты/банки» через веб-поиск при изменениях.
- **Росстат** (`rosstat.gov.ru` — проблема с сертификатом) / **Минфин** (`minfin.gov.ru` — 503): инфляция, бюджет, аукционы ОФЗ — раз в месяц/по событию, через поиск.

## Что отслеживать ПОД ЭТОТ портфель

На основании текущего состава портфеля выбирай сам и перед началом поиска объясняй свой выбор и утверждай приоритет с пользователем.

---

## Пример сборки дайджеста (как это делалось)

Веером, затем синтез:
1. **RSS-поток** (уровень 1) за период → отфильтровать по ключевым темам/тикерам выше.
2. **Веб-поиск:** ставка — дата след. заседания + консенсус; инфляция (последний ИПЦ + ожидания); аукционы Минфина.
3. **Веб-поиск:** геополитика (перемирие/эскалация) + нефть (Brent уровень/тренд, Urals/дисконт) + санкции.
4. **Веб-поиск по эмитентам:** дивиденды/отчёты/рейтинги/санкции по каждой бумаге.
5. **Синтез:** свести в `[ФАКТ]/[ОЖИДАНИЕ]/[МНЕНИЕ]`, дедуп, и для каждого блока — вывод по портфелю (цепочка из «Правил», п.6).

## Оговорки
- **Сеть:** Reuters/FT/Bloomberg блокируют автозагрузку; РФ-госсайты капризны (серт/503). Работай через веб-поиск, где прямой доступ не идёт.
- **Поляризация инфополя:** взвешивай источник, не принимай на веру; особый правовой статус части изданий учитывай в тоне и доступности.
